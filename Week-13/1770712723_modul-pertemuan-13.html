Apa itu Deployment?

Deployment adalah proses memindahkan aplikasi dari lingkungan development (localhost) ke server production agar dapat diakses oleh pengguna melalui internet.

Halaman 2

📦 PERSIAPAN DEPLOYMENT

1. Buat File Konfigurasi Environment

config/config.php <?php /** * Konfigurasi Aplikasi * Sesuaikan dengan environment (development/production) */ // Deteksi environment $isProduction = ($_SERVER['SERVER_NAME'] !== 'localhost'); if ($isProduction) { // ===== KONFIGURASI PRODUCTION ===== define('DB_HOST', 'localhost'); define('DB_NAME', 'username_dbakademik'); // Biasanya prefix_namadb define('DB_USER', 'username_dbuser'); // User dari cPanel define('DB_PASS', 'password_kuat_123'); define('BASE_URL', 'https://namadomain.com/'); // Matikan error display di production ini_set('display_errors', 0); error_reporting(0); } else { // ===== KONFIGURASI DEVELOPMENT ===== define('DB_HOST', 'localhost'); define('DB_NAME', 'db_akademik'); define('DB_USER', 'root'); define('DB_PASS', ''); define('BASE_URL', 'http://localhost/project-mvc/'); // Tampilkan error di development ini_set('display_errors', 1); error_reporting(E_ALL); } // Konfigurasi umum define('APP_NAME', 'Sistem Akademik'); define('UPLOAD_PATH', __DIR__ . '/../uploads/'); ?>
Penting! Jangan pernah commit password database ke Git. Gunakan file .env atau config.php yang di-ignore.
Halaman 3

2. Update Koneksi Database

config/database.php - Gunakan Konstanta <?php require_once __DIR__ . '/config.php'; class Database { private static $instance = null; private $pdo; private function __construct() { try { $dsn = "mysql:host=" . DB_HOST . ";dbname=" . DB_NAME . ";charset=utf8mb4"; $options = [ PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION, PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC, PDO::ATTR_EMULATE_PREPARES => false ]; $this->pdo = new PDO($dsn, DB_USER, DB_PASS, $options); } catch (PDOException $e) { // Di production, jangan tampilkan detail error if (defined('BASE_URL') && strpos(BASE_URL, 'localhost') === false) { die('Terjadi kesalahan sistem. Silakan coba lagi nanti.'); } die('Database Error: ' . $e->getMessage()); } } public static function getInstance() { if (self::$instance === null) { self::$instance = new self(); } return self::$instance; } public function getConnection() { return $this->pdo; } } function db() { return Database::getInstance()->getConnection(); } ?>
Keuntungan: Dengan konfigurasi terpisah, kode otomatis menyesuaikan apakah berjalan di localhost atau di server production.

🖥 MENGENAL cPANEL

cPanel adalah control panel yang paling umum digunakan di shared hosting. Berikut fitur-fitur penting yang akan kita gunakan:

FiturFungsiIcon
File Manager Upload, edit, hapus file langsung dari browser 📁
MySQL Databases Membuat database dan user database 🗃
phpMyAdmin Mengelola database (import, export, query) 📊
FTP Accounts Membuat akun FTP untuk upload via FileZilla 📤
SSL/TLS Mengaktifkan HTTPS untuk website 🔒
Domains Mengelola domain dan subdomain 🌐

Struktur Folder di Hosting

Struktur Folder cPanel /home/username/ ├── public_html/ # Document Root (folder utama website) │ ├── index.php # File utama yang diakses │ ├── config/ │ ├── models/ │ ├── controllers/ │ ├── views/ │ ├── includes/ │ ├── uploads/ │ └── public/ ├── logs/ # Error logs ├── mail/ # Email └── tmp/ # Temporary files
Document Root: Semua file website harus diletakkan di folder public_html agar bisa diakses via browser.
Halaman 6

🗃 MEMBUAT DATABASE DI HOSTING

1 Buka MySQL Databases di cPanel

Login ke cPanel, cari menu MySQL Databases atau MySQL Database Wizard.

2 Buat Database Baru

Masukkan nama database (contoh: akademik). Nama lengkap akan menjadi: username_akademik

3 Buat User Database

Buat user baru dengan password yang kuat. Nama lengkap: username_dbuser

4 Hubungkan User ke Database

Pilih user dan database, lalu berikan ALL PRIVILEGES.

5 Import Database via phpMyAdmin
  1. Buka phpMyAdmin dari cPanel
  2. Pilih database yang baru dibuat
  3. Klik tab Import
  4. Pilih file SQL yang di-export dari localhost
  5. Klik Go untuk import
Catat Kredensial:
  • Database: username_namadb
  • User: username_dbuser
  • Password: ********
  • Host: localhost (biasanya)

Halaman 8

🔧 KONFIGURASI SETELAH UPLOAD

1. Update config.php

Edit file config/config.php via File Manager dan sesuaikan kredensial database:

Konfigurasi Production // Bagian production di config.php define('DB_HOST', 'localhost'); define('DB_NAME', 'cpanelusr_akademik'); // Sesuaikan! define('DB_USER', 'cpanelusr_dbuser'); // Sesuaikan! define('DB_PASS', 'PasswordKuat123!'); // Sesuaikan! define('BASE_URL', 'https://namadomain.com/');

2. Set Permission Folder Uploads

Via File Manager # Klik kanan folder uploads > Change Permissions Permission: 755 # atau 777 jika masih error # Pastikan checkbox berikut tercentang: [x] Read [x] Write [x] Execute # Owner [x] Read [ ] Write [x] Execute # Group [x] Read [ ] Write [x] Execute # World

3. Cek .htaccess (Jika Perlu)

.htaccess (di public_html) # Redirect ke HTTPS (opsional) RewriteEngine On RewriteCond %{HTTPS} off RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301] # Blokir akses langsung ke folder sensitif RewriteRule ^config/ - [F,L] RewriteRule ^models/ - [F,L] RewriteRule ^controllers/ - [F,L] RewriteRule ^includes/ - [F,L] # Set default timezone (opsional) php_value date.timezone "Asia/Jakarta"

🚧 TROUBLESHOOTING

Error Umum & Solusinya

ErrorPenyebabSolusi
500 Internal Server Error Syntax error PHP atau .htaccess salah Cek error log, perbaiki syntax
Database connection failed Kredensial database salah Cek DB_NAME, DB_USER, DB_PASS
Access denied for user User tidak punya privilege Tambahkan user ke database di cPanel
Blank/white page Fatal error, display_errors off Aktifkan error sementara atau cek log
403 Forbidden Permission folder salah Set permission 755 atau 644
Upload gagal Folder tidak writable Set permission uploads ke 755/777
Gambar tidak muncul Path salah atau file tidak ada Cek path relatif/absolut
Session tidak bekerja session_start() tidak dipanggil Pastikan session_start() di awal

Cara Melihat Error Log

Lokasi Error Log di cPanel # Via File Manager: /home/username/logs/error.log # Atau via cPanel menu: Metrics > Errors # Untuk melihat error PHP secara sementara, tambahkan di awal file: ini_set('display_errors', 1); error_reporting(E_ALL);
Halaman 11

🔒 MENGAKTIFKAN SSL/HTTPS

Mengapa HTTPS Penting?
  • Enkripsi data antara browser dan server
  • Melindungi password dan data sensitif
  • SEO lebih baik (Google memprioritaskan HTTPS)
  • Kepercayaan pengguna meningkat

Mengaktifkan SSL Gratis (Let's Encrypt)

1 Buka SSL/TLS di cPanel

Cari menu SSL/TLS atau Let's Encrypt SSL

2 Pilih Domain

Pilih domain yang ingin diaktifkan SSL-nya

3 Install Certificate

Klik Issue atau Install. Tunggu proses selesai.

4 Force HTTPS via .htaccess
# Tambahkan di .htaccess RewriteEngine On RewriteCond %{HTTPS} off RewriteRule ^(.*)$ https://%{HTTP_HOST}%{REQUEST_URI} [L,R=301]
✅ Selesai! Website sekarang bisa diakses dengan https:// dan akan otomatis redirect dari http ke https.

Halaman 14

📋 TUGAS PRAKTIKUM

✅ Tugas: Deploy Project ke Hosting

Deploy project yang sudah dibuat ke hosting (gratis/berbayar):

  1. Daftar akun hosting (bisa gunakan 000webhost, InfinityFree, atau lainnya)
  2. Buat file konfigurasi environment (config.php)
  3. Buat database di hosting dan import SQL
  4. Upload semua file project ke public_html
  5. Konfigurasi kredensial database untuk production
  6. Set permission folder uploads
  7. Test semua fitur (CRUD, Login, Upload)
  8. Aktifkan SSL/HTTPS jika tersedia
  9. Kumpulkan URL website yang sudah online

Kriteria Penilaian

Database Berfungsi20%Data tampil dengan benar
