Apa itu Debugging?

Debugging adalah proses menemukan dan memperbaiki kesalahan (bug) dalam kode program. Istilah "bug" berasal dari serangga nyata yang ditemukan di dalam komputer pada tahun 1947.

🚨 JENIS-JENIS ERROR PHP

🚫 Fatal Error

Error kritis yang menghentikan eksekusi script. Contoh: memanggil fungsi yang tidak ada, class tidak ditemukan.

⚠️ Parse Error

Kesalahan syntax yang mencegah PHP mem-parse kode. Contoh: lupa titik koma, kurung tidak lengkap.

⚠️ Warning

Peringatan yang tidak menghentikan eksekusi. Contoh: include file yang tidak ada, division by zero.

📝 Notice

Pemberitahuan minor. Contoh: mengakses variabel yang belum didefinisikan, index array tidak ada.

Contoh Error Messages

Contoh Pesan Error Fatal error: Uncaught Error: Call to undefined function getUser() in /var/www/html/index.php on line 15 Warning: include(config.php): Failed to open stream: No such file or directory in /var/www/html/index.php on line 3 Parse error: syntax error, unexpected '}' in /var/www/html/index.php on line 25 Notice: Undefined variable: $username in /var/www/html/index.php on line 10
Tips Membaca Error:
  1. Perhatikan jenis error (Fatal, Warning, Notice)
  2. Baca deskripsi masalahnya
  3. Cek nama file dan nomor baris
  4. Buka file tersebut dan periksa baris yang disebutkan

Konfigurasi Error Reporting

Mengaktifkan Error Display <?php // ===== DI AWAL FILE UNTUK DEBUGGING ===== // Tampilkan semua error ini_set('display_errors', 1); ini_set('display_startup_errors', 1); error_reporting(E_ALL); // Atau hanya error tertentu error_reporting(E_ERROR | E_WARNING | E_PARSE); // Simpan error ke file log ini_set('log_errors', 1); ini_set('error_log', '/path/to/error.log'); ?>
Custom Error Handler <?php // Fungsi custom untuk menangani error function customErrorHandler($errno, $errstr, $errfile, $errline) { $message = "[{$errno}] {$errstr} in {$errfile} on line {$errline}"; // Log ke file error_log($message, 3, 'logs/error.log'); // Tampilkan pesan user-friendly if ($errno === E_USER_ERROR) { echo '<div class="alert alert-danger">Terjadi kesalahan sistem.</div>'; exit(1); } return true; } // Set custom handler set_error_handler('customErrorHandler'); ?>
Ingat! Selalu matikan display_errors di production dan gunakan error logging sebagai gantinya.

🔍 TEKNIK DEBUGGING PHP

1. var_dump() dan print_r()

Melihat Isi Variabel <?php $user = [ 'id' => 1, 'nama' => 'John Doe', 'email' => '[email protected]', 'active' => true ]; // var_dump - menampilkan tipe data dan nilai echo '<pre>'; var_dump($user); echo '</pre>'; // print_r - menampilkan struktur array/object echo '<pre>'; print_r($user); echo '</pre>'; // Atau dengan return value $debug = print_r($user, true); error_log($debug); // Simpan ke log ?>
Output var_dump() array(4) { ["id"]=> int(1) ["nama"]=> string(8) "John Doe" ["email"]=> string(14) "[email protected]" ["active"]=> bool(true) }

2. die() dan exit()

Menghentikan Eksekusi untuk Debug <?php // Debug: cek nilai sebelum proses $query = "SELECT * FROM users WHERE id = {$id}"; die("Query: " . $query); // Tampilkan dan stop // Atau dengan var_dump var_dump($_POST); exit; // Stop eksekusi // Debug di titik tertentu if ($condition) { die('Sampai sini OK'); } ?>

3. error_log() - Debug tanpa Mengganggu Output

Logging ke File <?php // Log pesan sederhana error_log('User login attempt: ' . $email); // Log array/object error_log(print_r($_POST, true)); // Log ke file khusus error_log( date('Y-m-d H:i:s') . ' - Debug: ' . $message . "\n", 3, 'logs/debug.log' ); // Helper function untuk debugging function debug_log($data, $label = 'DEBUG') { $output = is_array($data) || is_object($data) ? print_r($data, true) : $data; error_log("[{$label}] {$output}"); } // Penggunaan debug_log($_SESSION, 'SESSION DATA'); debug_log($result, 'QUERY RESULT'); ?>

4. Debugging Database Query

Debug Query PDO <?php // Aktifkan error mode $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION); try { $sql = "SELECT * FROM mahasiswa WHERE nim = :nim"; $stmt = $pdo->prepare($sql); $stmt->execute([':nim' => $nim]); // Debug: lihat query yang dieksekusi error_log("Query: {$sql}, Param: {$nim}"); $result = $stmt->fetchAll(); // Debug: cek hasil error_log("Rows found: " . count($result)); } catch (PDOException $e) { // Log error database error_log("DB Error: " . $e->getMessage()); // Debug: tampilkan di development if ($_SERVER['SERVER_NAME'] === 'localhost') { echo "<pre>Error: " . $e->getMessage() . "</pre>"; } } ?>

Debug Form Submit dengan Network Tab

1 Buka Tab Network

Tekan F12, klik tab Network, centang Preserve log

2 Submit Form

Isi form dan klik submit. Akan muncul request baru di list.

3 Klik Request

Contoh Info di Network Tab Request URL: http://localhost/project/proses.php Request Method: POST Status Code: 200 OK atau 500 Internal Server Error Form Data: nim: 12345678 nama: John Doe email: [email protected] Response: <!-- Jika error, response bisa berisi pesan error PHP --> {"success": true, "message": "Data berhasil disimpan"}
Tips: Jika status 500, lihat tab Response untuk melihat pesan error PHP yang mungkin tersembunyi.
Halaman 7

🐛 BUG UMUM & SOLUSINYA

🐛

Undefined Variable / Index

Penyebab: Mengakses variabel atau array key yang belum ada
Solusi: Gunakan isset() atau null coalescing ??
$name = $_POST['name'] ?? '';
🐛

Headers Already Sent

Penyebab: Ada output (spasi, HTML) sebelum header() atau session_start()
Solusi: Pastikan tidak ada karakter apapun sebelum <?php, termasuk BOM character. Letakkan session_start() di paling awal.
🐛

Class Not Found

Penyebab: File class tidak di-include atau path salah
Solusi: Cek path require_once, gunakan __DIR__ untuk path relatif.
require_once __DIR__ . '/../models/User.php';
🐛

SQLSTATE Connection Refused

Penyebab: Database tidak berjalan atau kredensial salah
Solusi: Cek MySQL/XAMPP running, verifikasi DB_HOST, DB_USER, DB_PASS
Halaman 8

Bug Umum Lainnya

🐛

Form Data Tidak Terkirim

Penyebab: Lupa name attribute, method salah, atau enctype untuk file
Solusi:
- Pastikan setiap input punya name="..."
- Gunakan method="POST" untuk form
- Tambah enctype="multipart/form-data" untuk upload file
🐛

Session Tidak Tersimpan

Penyebab: session_start() tidak dipanggil atau dipanggil setelah output
Solusi: Letakkan session_start() di baris pertama sebelum output apapun
🐛

Gambar/CSS Tidak Muncul

Penyebab: Path relatif salah, terutama setelah deploy
Solusi: Gunakan path dari root dengan / atau BASE_URL
<img src="<?= BASE_URL ?>uploads/foto.jpg">
🐛

Redirect Loop

Penyebab: Kondisi redirect yang salah sehingga terus menerus redirect
Solusi: Cek kondisi requireLogin(), pastikan halaman login tidak di-protect

✅ TESTING MANUAL

Testing Manual adalah proses menguji aplikasi secara langsung oleh manusia tanpa menggunakan tools otomatis. Ini penting untuk memastikan aplikasi berfungsi dari perspektif pengguna.

Test Case Template

TC-001: Login dengan kredensial valid PASS
Langkah:
  1. Buka halaman login
  2. Masukkan email: [email protected]
  3. Masukkan password: password123
  4. Klik tombol Login
Expected: Redirect ke dashboard, nama user muncul di navbar
Actual: Sesuai expected
TC-002: Login dengan password salah PASS
Langkah:
  1. Buka halaman login
  2. Masukkan email: [email protected]
  3. Masukkan password: wrongpassword
  4. Klik tombol Login
Expected: Tampil pesan error "Email atau password salah"
Actual: Sesuai expected
TC-003: Upload file melebihi batas FAIL
Langkah:
  1. Buka form tambah mahasiswa
  2. Upload file gambar 5MB (batas 2MB)
  3. Klik Simpan
Expected: Tampil pesan error "Ukuran file maksimal 2MB"
Actual: Blank page, tidak ada pesan error

💡 BEST PRACTICES

Tips Debugging Efektif:
  1. Baca error message dengan teliti - Informasi penting ada di sana
  2. Reproduksi bug secara konsisten - Pastikan bisa mengulang bug
  3. Isolasi masalah - Cari tahu bagian mana yang bermasalah
  4. Cek perubahan terakhir - Apa yang baru diubah?
  5. Gunakan version control - Git membantu melacak perubahan
  6. Minta bantuan - Fresh eyes bisa menemukan bug lebih cepat
Hal yang Harus Dihindari:
  • Debugging di production tanpa backup
  • Mengubah banyak hal sekaligus tanpa test
  • Mengabaikan warning dan notice
  • Tidak mencatat bug yang ditemukan
  • Lupa menghapus kode debug sebelum commit

Debug Helper Function

includes/debug.php <?php /** * Helper functions untuk debugging * HAPUS atau nonaktifkan di production! */ // Pretty print untuk debugging function dd($data) { echo '<pre style="background:#1a1a2e;color:#e2e8f0;padding:15px;border-radius:8px;">'; var_dump($data); echo '</pre>'; exit; } // Dump tanpa exit function dump($data) { echo '<pre style="background:#1a1a2e;color:#e2e8f0;padding:15px;border-radius:8px;">'; var_dump($data); echo '</pre>'; } // Log ke file function logDebug($message, $data = null) { $log = date('Y-m-d H:i:s') . " - {$message}"; if ($data) { $log .= ": " . print_r($data, true); } file_put_contents('logs/debug.log', $log . "\n", FILE_APPEND); } ?>

📋 TUGAS PRAKTIKUM

✅ Tugas: Testing & Bug Fixing

Lakukan testing pada project yang sudah di-deploy dan dokumentasikan hasilnya:

  1. Buat minimal 10 test case untuk fitur utama aplikasi
  2. Jalankan semua test case dan catat hasilnya (PASS/FAIL)
  3. Untuk setiap test yang FAIL, buat bug report
  4. Perbaiki bug yang ditemukan
  5. Jalankan ulang test untuk memastikan bug sudah diperbaiki
  6. Lakukan testing keamanan dasar (SQL Injection, XSS)
  7. Dokumentasikan semua proses dalam laporan testing

Kriteria Penilaian

AspekBobotKriteria
Test Cases25%Minimal 10 test case lengkap
Bug Reports20%Format lengkap, reproducible
Bug Fixing25%Bug berhasil diperbaiki
Security Testing15%Test SQL Injection & XSS
Dokumentasi15%Laporan rapi dan lengkap
🚧 Format Pengumpulan:

Kumpulkan laporan testing dalam format PDF yang berisi: daftar test case, hasil testing, bug reports, dan bukti perbaikan (screenshot before/after).
