🎨 BAGIAN 1: PENGANTAR UI DESIGN

Apa itu User Interface (UI)?

User Interface (UI) adalah segala sesuatu yang dirancang agar pengguna dapat berinteraksi dengan sebuah produk digital. UI mencakup tampilan visual, elemen interaktif, tata letak, warna, tipografi, ikon, dan tombol yang membentuk pengalaman visual pengguna saat menggunakan website atau aplikasi.

Perbedaan UI dan UX

Aspek UI (User Interface) UX (User Experience)
Fokus Tampilan visual dan estetika Pengalaman keseluruhan pengguna
Pertanyaan "Apakah tampilannya menarik?" "Apakah mudah digunakan?"
Mencakup Warna, font, tombol, ikon, layout Alur navigasi, kecepatan, aksesibilitas, kepuasan
Analogi Seperti cat, dekorasi, dan furnitur di rumah Seperti denah, struktur, dan kenyamanan rumah
Tools Figma, Canva, Adobe XD Flowchart, User Journey Map, Usability Test
Hubungan UI dan UX: UI yang baik tanpa UX yang baik ibarat rumah cantik tapi susah dinavigasi. UX yang baik tanpa UI yang baik ibarat rumah nyaman tapi kurang menarik secara visual. Keduanya harus berjalan beriringan.
Halaman 2

💡 BAGIAN 2: PRINSIP-PRINSIP DESAIN UI

Terdapat beberapa prinsip fundamental yang harus dipahami untuk menciptakan desain antarmuka yang baik:

1⃣

Konsistensi (Consistency)

Elemen desain harus konsisten di seluruh halaman. Jika tombol utama berwarna biru, maka semua tombol utama di setiap halaman harus berwarna biru. Font, ukuran teks, warna, dan jarak antar elemen harus seragam.

2⃣

Hierarki Visual (Visual Hierarchy)

Atur elemen berdasarkan tingkat kepentingan. Elemen penting dibuat lebih besar, lebih tebal, atau berwarna mencolok. Pengguna harus langsung tahu apa yang paling penting di halaman tersebut.

3⃣

Kesederhanaan (Simplicity)

Desain yang baik adalah desain yang sederhana. Hindari elemen yang tidak perlu. Setiap elemen harus memiliki tujuan yang jelas. Prinsip: "Less is more".

4⃣

Kontras (Contrast)

Gunakan perbedaan warna, ukuran, atau bentuk untuk membedakan elemen. Teks harus mudah dibaca dengan kontras yang cukup terhadap latar belakangnya. Tombol penting harus menonjol dari elemen lain.

5⃣

Alignment (Perataan)

Setiap elemen harus memiliki koneksi visual dengan elemen lain. Gunakan grid system untuk menjaga elemen tetap teratur dan rapi. Jangan menempatkan elemen secara acak.

6⃣

Proximity (Kedekatan)

Elemen yang saling berhubungan harus diletakkan berdekatan. Label dan input form harus dekat. Grup informasi yang terkait harus jelas terpisah dari grup lainnya.

7⃣

Feedback (Umpan Balik)

Sistem harus memberikan respon terhadap aksi pengguna. Contoh: tombol berubah warna saat di-hover, loading indicator saat proses berjalan, pesan sukses setelah form disubmit.

Teks ini sangat jelas dan mudah dibaca dengan kontras yang baik.

Klik Di Sini

Kontras tinggi, teks gelap di latar terang. Tombol berwarna mencolok, jelas bisa diklik.

🎨 BAGIAN 4: WARNA & TIPOGRAFI

Teori Warna untuk UI

Pemilihan warna sangat berpengaruh terhadap kesan dan pengalaman pengguna. Setiap warna memiliki makna psikologis:

Warna Contoh Kesan / Makna Penggunaan Umum
Biru Kepercayaan, profesional, tenang Facebook, LinkedIn, perbankan
Merah Urgensi, energi, bahaya Tombol hapus, notifikasi error, promo
Hijau Sukses, alam, pertumbuhan Pesan sukses, tombol konfirmasi, Go-Jek
Kuning/Oranye Perhatian, optimis, hangat Warning, highlight, Shopee
Putih/Abu-abu Bersih, netral, minimalis Background, space, Apple

Tipografi untuk Web

Heading 1 - 28px Bold

Digunakan untuk judul utama halaman

Heading 2 - 22px Semibold

Digunakan untuk sub-judul / section title

Heading 3 - 18px Semibold

Digunakan untuk judul card atau komponen

Body Text - 16px Regular

Digunakan untuk teks paragraf utama

Small / Caption - 13px Regular

Digunakan untuk keterangan, label, footnote

Tips Tipografi: Gunakan maksimal 2 jenis font dalam satu desain (1 untuk heading, 1 untuk body). Font yang umum untuk web: Inter, Roboto, Open Sans, Poppins (sans-serif) atau Merriweather, Lora (serif).

Struktur Layout Standar:

  • Header/Navbar: Logo + menu navigasi. Biasanya fixed di atas halaman
  • Hero Section: Area utama berisi headline, tagline, dan tombol CTA (Call to Action)
  • Content Area: Konten utama halaman (card, artikel, tabel, form, dll.)

  • Footer: Informasi tambahan, copyright, link sosial media

📝 BAGIAN 6: WIREFRAME & MOCKUP

Apa itu Wireframe?

Wireframe adalah sketsa sederhana (low-fidelity) dari tata letak halaman web yang menunjukkan posisi elemen-elemen utama tanpa detail visual (tanpa warna, gambar, atau font spesifik). Wireframe fokus pada struktur dan fungsi, bukan estetika.

Tools untuk Membuat Wireframe & Mockup

Tool Jenis Kelebihan Akses

Rekomendasi untuk Praktikum: Pada praktikum ini kita akan menggunakan Figma sebagai alat utama karena gratis, berbasis browser (tidak perlu install), dan merupakan standar industri saat ini. Buka figma.com dan daftar dengan akun Google/email.

Pengenalan Figma

Figma adalah design tool berbasis browser yang sangat populer di kalangan UI/UX designer. Berikut area kerja utama Figma:

1
Membuat Akun & Project Baru
  • Buka figma.com di browser komputer lab
  • Sign up / Login dengan akun Google
  • Klik "New design file" untuk membuat file baru
2
Mengenal Area Kerja Figma
  • Toolbar (atas): Alat-alat desain (Frame, Shape, Text, Hand, dll.)
  • Left Panel: Layers & Assets - mengelola elemen desain
  • Canvas (tengah): Area kerja utama tempat mendesain
  • Right Panel: Properties - mengatur warna, ukuran, font, dll.
3
Tools Dasar Figma
  • Frame (F): Membuat artboard/frame (layar/halaman)
  • Rectangle (R): Membuat kotak/persegi
  • Ellipse (O): Membuat lingkaran/oval
  • Text (T): Menambahkan teks
  • Line (L): Membuat garis
  • Hand (H): Menggeser canvas
  • Zoom: Ctrl + scroll untuk zoom in/out

💻 BAGIAN 8: PRAKTIK - MEMBUAT WIREFRAME DI FIGMA

Studi Kasus: Wireframe Halaman Login Sistem Akademik

Ikuti langkah-langkah berikut untuk membuat wireframe halaman login:

1
Buat Frame

Tekan F atau pilih Frame tool, lalu pilih preset "Desktop" (1440 x 1024) di panel kanan. Ini akan menjadi kanvas halaman login kita.

2
Buat Background

Klik frame, lalu di panel kanan ubah warna Fill menjadi #F8F9FA (abu-abu muda) sebagai background.

3
Buat Card Login (tengah)
  • Tekan R untuk Rectangle
  • Buat kotak ukuran 400 x 450 pixel
  • Fill: #FFFFFF (putih), Corner Radius: 12
  • Tambahkan shadow: Effect → Drop Shadow (X:0, Y:4, Blur:20, Opacity:10%)
  • Posisikan di tengah frame
4
Tambahkan Elemen Form
  • Tekan T untuk Text → ketik "Login" (24px, Bold, #2a5298)
  • Tekan T → ketik "Email" (14px, Medium, #333)
  • Tekan R → buat input field (350 x 44, Fill: #F0F0F0, Corner: 8)
  • Ulangi untuk label "Password" dan input field-nya
  • Buat tombol: Rectangle (350 x 44, Fill: #2a5298, Corner: 8)
  • Tambah teks di tombol: "Masuk" (16px, Bold, White)
5
Tambahkan Detail
  • Logo/judul di atas card: "Sistem Akademik"
  • Link di bawah tombol: "Belum punya akun? Daftar" (13px, #2a5298)
  • Placeholder text di input: "masukkan email..." (14px, #AAAAAA)

Latihan Praktikum

1
Latihan 1: Identifikasi Prinsip Desain

Buka 3 website populer (misalnya: Tokopedia, Instagram, Google). Identifikasi dan tuliskan mana prinsip desain UI yang diterapkan pada masing-masing website tersebut (minimal 3 prinsip per website).

2
Latihan 2: Wireframe Halaman Login
