📄 BAGIAN 1: PENGANTAR HTML

Apa itu HTML?

HTML (HyperText Markup Language) adalah bahasa markup standar untuk membuat struktur halaman web. HTML bukanlah bahasa pemrograman, melainkan bahasa yang memberikan "tanda" (markup) pada konten agar browser tahu cara menampilkannya.

Konsep Dasar HTML:

  • Tag: Penanda yang mengapit konten, contoh: <p>...</p>
  • Element: Tag pembuka + konten + tag penutup
  • Attribute: Informasi tambahan pada tag, contoh: class="nama"
  • Nesting: Tag di dalam tag (bersarang)

Struktur Dasar Dokumen HTML5

Setiap halaman HTML memiliki struktur dasar yang harus diikuti:

index.html <!-- Deklarasi tipe dokumen HTML5 --> <!DOCTYPE html> <html lang="id"> <head> <!-- Meta informasi halaman --> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Judul Halaman</title> <!-- Link ke file CSS --> <link rel="stylesheet" href="style.css"> </head> <body> <!-- Konten halaman di sini --> <h1>Selamat Datang</h1> <p>Ini adalah paragraf pertama.</p> </body> </html>
BagianFungsi
<!DOCTYPE html>Deklarasi bahwa dokumen ini adalah HTML5
<html>Elemen root yang membungkus seluruh halaman
<head>Informasi meta, title, link CSS (tidak tampil di halaman)
<body>Konten yang tampil di browser

HTML Semantik

HTML5 memperkenalkan tag semantik yang memberikan makna pada struktur halaman:

Non-Semantik (Lama) <div id="header">...</div> <div id="nav">...</div> <div id="main">...</div> <div id="footer">...</div>
Semantik (Modern) ✅ <header>...</header> <nav>...</nav> <main>...</main> <footer>...</footer>
Tag SemantikFungsi
<header>Bagian header halaman/section
<nav>Navigasi utama
<main>Konten utama halaman
<section>Section/bagian konten
<article>Konten independen (artikel, post)
<aside>Sidebar atau konten sampingan
<footer>Bagian footer
Mengapa Semantik Penting? Tag semantik membantu: 1) SEO lebih baik, 2) Aksesibilitas (screen reader), 3) Kode lebih mudah dibaca dan dimaintain.

Form HTML - Penting untuk CRUD!

Form adalah elemen krusial dalam aplikasi web karena digunakan untuk menerima input dari pengguna. Nantinya form ini akan memproses data ke PHP.

Contoh Form Lengkap <form action="proses.php" method="POST"> <!-- Input Text --> <label for="nama">Nama:</label> <input type="text" id="nama" name="nama" required> <!-- Input Email --> <label for="email">Email:</label> <input type="email" id="email" name="email" required> <!-- Input Password --> <label for="password">Password:</label> <input type="password" id="password" name="password"> <!-- Select/Dropdown --> <label for="prodi">Program Studi:</label> <select id="prodi" name="prodi"> <option value="">-- Pilih --</option> <option value="TI">Teknologi Informasi</option> <option value="SI">Sistem Informasi</option> </select> <!-- Textarea --> <label for="alamat">Alamat:</label> <textarea id="alamat" name="alamat" rows="3"></textarea> <!-- Tombol Submit --> <button type="submit">Simpan Data</button> </form>
Atribut FormFungsi
actionURL tujuan pengiriman data (file PHP yang akan memproses)
methodGET (data di URL) atau POST (data tersembunyi)
nameNama field, akan menjadi key di PHP ($_POST['nama'])
idIdentifier unik, digunakan untuk label dan CSS/JS
requiredField wajib diisi
placeholderTeks petunjuk di dalam input
Penting untuk Project: Atribut name pada setiap input HARUS ada karena akan digunakan untuk mengambil data di PHP dengan $_POST['nama_field'] atau $_GET['nama_field'].

🎨 BAGIAN 2: PENGANTAR CSS

Apa itu CSS?

CSS (Cascading Style Sheets) adalah bahasa yang digunakan untuk mengatur tampilan visual elemen HTML. CSS memisahkan konten (HTML) dari presentasi (tampilan).

3 Cara Menambahkan CSS:

1. Inline CSS - langsung di atribut style tag
<p style="color: red;">Teks merah</p>
2. Internal CSS - di dalam tag <style> di head
<style> p { color: red; } </style>
3. External CSS ✅ (Rekomendasi) - file terpisah
<link rel="stylesheet" href="style.css">

Sintaks CSS

Struktur Dasar CSS selector { property: value; property: value; } /* Contoh: */ h1 { color: #1d976c; font-size: 24px; margin-bottom: 15px; } p { color: #333; line-height: 1.6; }

CSS Selector

SelectorContohMemilih
Elementp { }Semua elemen <p>
Class.box { }Elemen dengan class="box"
ID#header { }Elemen dengan id="header"
Descendantnav a { }Semua <a> di dalam <nav>
Childul > li { }<li> yang langsung di dalam <ul>
Pseudo-classa:hover { }<a> saat mouse di atasnya
Multipleh1, h2, h3 { }h1, h2, dan h3 sekaligus
Best Practice: Gunakan class untuk styling yang bisa dipakai ulang. Gunakan id hanya untuk elemen unik (biasanya untuk JavaScript). Hindari inline CSS.

Layout dengan Flexbox

Flexbox adalah metode layout CSS modern yang memudahkan pengaturan posisi dan ukuran elemen dalam satu baris atau kolom.

Flexbox Dasar /* Container flex */ .container { display: flex; justify-content: space-between; /* Horizontal */ align-items: center; /* Vertical */ gap: 20px; /* Jarak antar item */ } /* Item di dalam flex container */ .item { flex: 1; /* Semua item ukuran sama */ }
PropertyValueFungsi
justify-contentflex-start, flex-end, centerPosisi horizontal
space-between, space-aroundDistribusi ruang
align-itemsflex-start, center, flex-end, stretchPosisi vertikal
flex-directionrow, columnArah item (baris/kolom)
flex-wrapnowrap, wrapWrap ke baris baru
gap10px, 20pxJarak antar item

📋 EVALUASI & TUGAS KELOMPOK

👥 Tugas Project Kelompok - Pertemuan 3

Setiap kelompok (3-4 orang) harus membuat halaman-halaman HTML + CSS berikut untuk project Sistem Informasi:

  1. index.html - Halaman Login (sudah dikerjakan bersama)
  2. dashboard.html - Halaman Dashboard setelah login
  3. mahasiswa.html - Halaman Daftar Data (tabel + tombol aksi)
  4. tambah.html - Form Tambah Data baru
  5. edit.html - Form Edit Data (mirip tambah)
  6. css/style.css - File CSS lengkap untuk semua halaman
Ketentuan Tugas:
  • Desain HARUS mengikuti wireframe yang dibuat di Pertemuan 2
  • Gunakan HTML semantik (header, nav, main, footer)
  • Gunakan External CSS (file terpisah)
  • Setiap form harus memiliki atribut name pada setiap input
  • Tampilan harus konsisten di semua halaman (navbar, warna, font sama)
