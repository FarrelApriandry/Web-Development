Halaman 3

Query Membuat Tabel Users

Tabel users digunakan untuk menyimpan data login (email, password, role).

SQL - Tabel Users -- Tabel untuk menyimpan data user login CREATE TABLE users ( id INT AUTO_INCREMENT PRIMARY KEY, email VARCHAR(100) NOT NULL UNIQUE, password VARCHAR(255) NOT NULL, nama VARCHAR(100) NOT NULL, role ENUM('admin', 'user') DEFAULT 'user', created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );

Query Membuat Tabel Mahasiswa

Tabel mahasiswa menyimpan data mahasiswa yang akan ditampilkan dan dikelola.

SQL - Tabel Mahasiswa -- Tabel untuk menyimpan data mahasiswa CREATE TABLE mahasiswa ( nim VARCHAR(10) PRIMARY KEY, nama VARCHAR(100) NOT NULL, email VARCHAR(100) UNIQUE, prodi VARCHAR(50), semester INT DEFAULT 1, alamat TEXT, no_hp VARCHAR(15), created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP );
Penjelasan Tipe Data:
  • INT - Bilangan bulat
  • VARCHAR(n) - Teks dengan panjang maksimal n karakter
  • TEXT - Teks panjang (untuk alamat)
  • ENUM - Pilihan terbatas (admin/user)
  • TIMESTAMP - Waktu otomatis saat data dibuat

Ringkasan Perbedaan Penting

Aspek index.html (Lama) index.php (Baru)
Session ❌ Tidak ada ✅ session_start() di awal
Form Action ❌ Kosong / JS only ✅ action="proses_login.php"
Form Method ❌ Tidak ada ✅ method="POST"
Input Name ❌ Hanya id ✅ Ada name="email"
Error Message ❌ Alert JS ✅ PHP dari $_GET['error']
Cek Login ❌ Tidak ada ✅ Redirect jika sudah login
💡 Penting untuk Diingat:
  • Atribut name pada input WAJIB ada agar data bisa dikirim via POST/GET
  • Atribut id tetap diperlukan untuk JavaScript (validasi client-side)
  • session_start() harus dipanggil sebelum output HTML apapun
  • Gunakan htmlspecialchars() untuk mencegah XSS attack
Validasi Tetap 2 Layer:
  1. Client-side (JavaScript): validasi.js tetap berjalan untuk UX yang baik
  2. Server-side (PHP): proses_login.php validasi ulang untuk keamanan

🔗 BAGIAN 1: KONEKSI DATABASE

Membuat File Koneksi dengan PDO

PDO (PHP Data Objects) adalah cara modern dan aman untuk terhubung ke database. PDO mendukung prepared statements yang mencegah SQL Injection.

config/database.php <?php /** * Konfigurasi Koneksi Database * File ini akan di-include di semua halaman yang butuh akses database */ // Konfigurasi Database $host = "localhost"; // Server database $dbname = "db_akademik"; // Nama database $username = "root"; // Username (default XAMPP) $password = ""; // Password (default XAMPP kosong) try { // Membuat koneksi PDO $pdo = new PDO( "mysql:host=$host;dbname=$dbname;charset=utf8", $username, $password ); // Set error mode ke exception $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION); // Set default fetch mode ke associative array $pdo->setAttribute(PDO::ATTR_DEFAULT_FETCH_MODE, PDO::FETCH_ASSOC); } catch (PDOException $e) { // Jika koneksi gagal die("Koneksi database gagal: " . $e->getMessage()); } ?>
Mengapa PDO?
  • Keamanan: Prepared statements mencegah SQL Injection
  • Fleksibel: Bisa digunakan untuk MySQL, PostgreSQL, SQLite, dll.
  • Modern: Standar yang direkomendasikan untuk PHP modern

Cara Menggunakan Koneksi

Contoh: Menggunakan koneksi di halaman lain <?php // Include file koneksi di awal halaman require_once 'config/database.php'; // Sekarang variabel $pdo bisa digunakan untuk query $stmt = $pdo->query("SELECT * FROM mahasiswa"); $data = $stmt->fetchAll(); foreach ($data as $row) { echo $row['nama'] . "<br>"; } ?>

File Proses Login

File ini menerima data dari form, memvalidasi, dan mengecek ke database.

proses_login.php <?php // Mulai session untuk menyimpan data login session_start(); // Include koneksi database require_once 'config/database.php'; // Cek apakah form di-submit dengan method POST if ($_SERVER['REQUEST_METHOD'] === 'POST') { // Ambil data dari form $email = trim($_POST['email']); $password = $_POST['password']; // Validasi: pastikan tidak kosong if (empty($email) || empty($password)) { header("Location: index.php?error=Email dan password wajib diisi!"); exit; } // Query ke database dengan prepared statement $sql = "SELECT * FROM users WHERE email = ?"; $stmt = $pdo->prepare($sql); $stmt->execute([$email]); $user = $stmt->fetch(); // Cek apakah user ditemukan dan password cocok if ($user && $password === $user['password']) { // Login berhasil! Simpan data ke session $_SESSION['user_id'] = $user['id']; $_SESSION['user_nama'] = $user['nama']; $_SESSION['user_email'] = $user['email']; $_SESSION['user_role'] = $user['role']; // Redirect ke dashboard header("Location: dashboard.php"); exit; } else { // Login gagal header("Location: index.php?error=Email atau password salah!"); exit; } } else { // Jika diakses langsung tanpa POST, redirect ke login header("Location: index.php"); exit; } ?>
Catatan Keamanan: Dalam production, password harus di-hash dengan password_hash() dan diverifikasi dengan password_verify(). Contoh di atas disederhanakan untuk pembelajaran.

Memahami Session

Session adalah cara untuk menyimpan data user antar halaman. Setelah login, data user disimpan di session sehingga kita tahu siapa yang sedang login.

Cara Kerja Session <?php // WAJIB di awal setiap halaman yang pakai session session_start(); // Menyimpan data ke session $_SESSION['user_id'] = 1; $_SESSION['user_nama'] = "Ahmad Rizki"; // Mengambil data dari session echo $_SESSION['user_nama']; // Ahmad Rizki // Cek apakah session ada if (isset($_SESSION['user_id'])) { echo "User sudah login"; } // Menghapus session (logout) session_destroy(); ?>

File Cek Login (Auth Guard)

includes/auth.php <?php /** * File ini di-include di halaman yang butuh login * Jika belum login, redirect ke halaman login */ session_start(); function cekLogin() { if (!isset($_SESSION['user_id'])) { header("Location: index.php?error=Silakan login terlebih dahulu!"); exit; } } function getUserNama() { return $_SESSION['user_nama'] ?? 'Guest'; } function getUserRole() { return $_SESSION['user_role'] ?? 'user'; } ?>

File Logout

logout.php <?php session_start(); // Hapus semua data session $_SESSION = []; // Hancurkan session session_destroy(); // Redirect ke halaman login header("Location: index.php?msg=Anda berhasil logout"); exit; ?>

Halaman 15

File Proses Tambah Data

proses_tambah.php <?php require_once 'includes/auth.php'; cekLogin(); require_once 'config/database.php'; if ($_SERVER['REQUEST_METHOD'] === 'POST') { // Ambil dan bersihkan data dari form $nim = trim($_POST['nim']); $nama = trim($_POST['nama']); $email = trim($_POST['email']); $prodi = $_POST['prodi']; $semester = (int)$_POST['semester']; $no_hp = trim($_POST['no_hp']); $alamat = trim($_POST['alamat']); // Validasi sederhana if (empty($nim) || empty($nama) || empty($email)) { header("Location: tambah.php?error=Data wajib harus diisi!"); exit; } try { // Prepared statement untuk INSERT (aman dari SQL Injection) $sql = "INSERT INTO mahasiswa (nim, nama, email, prodi, semester, no_hp, alamat) VALUES (?, ?, ?, ?, ?, ?, ?)"; $stmt = $pdo->prepare($sql); $stmt->execute([$nim, $nama, $email, $prodi, $semester, $no_hp, $alamat]); // Berhasil! Redirect ke halaman daftar header("Location: mahasiswa.php?success=Data berhasil ditambahkan!"); exit; } catch (PDOException $e) { // Jika NIM sudah ada (duplicate key) if ($e->getCode() == 23000) { header("Location: tambah.php?error=NIM sudah terdaftar!"); } else { header("Location: tambah.php?error=Terjadi kesalahan: " . $e->getMessage()); } exit; } } else { header("Location: tambah.php"); exit; } ?>
Prepared Statement: Dengan menggunakan ? sebagai placeholder dan execute() dengan array, data secara otomatis di-escape sehingga aman dari SQL Injection!

🧩 Tugas: Menyatukan Puzzle!

Setiap kelompok harus mengimplementasikan:

  1. config/database.php - Koneksi PDO ke database
  2. Sistem Login - index.php + proses_login.php
  3. includes/auth.php - Fungsi cek login
  4. dashboard.php - Dengan statistik dari database
  5. mahasiswa.php - Menampilkan data dari database
  6. tambah.php + proses_tambah.php - Form & proses tambah data
  7. logout.php - Fungsi logout
Checklist Sebelum Kumpul:
  • ☐ Database db_akademik sudah dibuat
  • ☐ Tabel users dan mahasiswa ada datanya
  • ☐ Login berfungsi dengan benar
  • ☐ Dashboard menampilkan statistik
  • ☐ Data mahasiswa ditampilkan dari database
  • ☐ Form tambah bisa menyimpan ke database
  • ☐ Logout berfungsi
